## 1️⃣ ROI / การเงิน 

**Q1: ตัวเลขประหยัดคิดจากอะไร?**
**A:** จากเวลาทำงานจริง ลดจาก 20 นาทีเหลือ 5 นาทีต่อเคส คิดแบบ Conservative

**Q1: ตัวเลข Volume, Time และ Cost นี้ “ของจริง” แค่ไหน?**
**A:** 
ตัวเลข 1,000 เคสต่อวัน กับเวลา 20 นาทีต่อเคส เป็นข้อมูลจริงหรือประมาณการ 
ของทีม Claim ครับ
และเวลา 20 นาทีต่อเคสเป็นค่าเฉลี่ยจากการทำงานจริงของเจ้าหน้าที่ 
เพราะฉะนั้น Productivity และเงินที่ประหยัดได้
สะท้อนสิ่งที่เกิดขึ้นในชีวิตจริงของธุรกิจครับ

**เรามั่นใจได้อย่างไรว่าตัวเลข ROI 9.4 ล้านต่อปีเกิดขึ้นจริง?**
**คำตอบ:**
9.4ล้าน เป็นค่าแรงเป็นค่าแรงที่คิดจาก 300/
และเป็นงานที่เกิดขึ้นจริง 1,000 เคสต่อวัน
ไม่ใช่งานคาดการณ์
เมื่อเวลาต่อเคสลดลง ตัวเลขต้นทุนจะลดทันที
เป็นประโยชน์ที่จับต้องได้ ไม่ใช่ Soft Benefit ครับ”


**Q2: ตัวเลขนี้เชื่อถือได้แค่ไหน?**
**A:** เป็น Base Case และขอวัดผลจริงใน Pilot ก่อนขยาย

**Q3: ลงทุนเท่าไหร่ คืนทุนกี่เดือน?**
**A:** Claim Use Case เดียว คืนทุนในปีแรก

**Q4: ถ้าใช้ OCR ไม่ถึง 50% จะยังคุ้มไหม?**
**A:** ยังประหยัดได้ตามสัดส่วน เริ่มเล็กแล้วค่อยขยาย

**Q5: รวม Error / Rework หรือยัง?**
**A:** ยังไม่รวม ตัวเลขนี้เฉพาะ Time Saving

## 2️⃣ Risk / Compliance / PDPA

**Q6: AI อ่านผิด ใครรับผิดชอบ?**
**A:** ระบบออกแบบแบบ Human-in-the-loop → AI เป็นแค่ผู้ช่วย คนยังคงเป็นผู้ตรวจทานก่อน

**มั่นใจได้แค่ไหน? กังวลเรื่องอ่านผิด**
**คำตอบ:**
“เรามี Human‑verification ครับ
AI อ่าน → คนตรวจจุดสำคัญ → เข้าระบบ
เพราะฉะนั้นความเสี่ยงเรื่องผิดพลาดจะถูกควบคุม
และเคสที่ OCR ไม่มั่นใจจะส่งไปตรวจโดยเจ้าหน้าที่
ไม่มีเคสที่ผิดหลุดไปอย่างไม่มีการตรวจครับ”

**Q7: PDPA มีความเสี่ยงไหม?**
**A:** 
ข้อมูลไม่ถูกส่งออกนอกองค์กร
ประมวลผลภายใน
มี Log การเข้าถึง และไม่เก็บข้อมูลเกินความจำเป็น
จึงไม่มีความเสี่ยงด้าน PDPA ครับ

**Q8: Audit ยังตรวจสอบได้ไหม?**
**A:** ได้ ทุกขั้นตอนมี Log และตรวจย้อนหลังได้

## 3️⃣ Operation / หน้างาน

**Q9: เอกสารแต่ละศูนย์ไม่เหมือนกัน จะอ่านได้จริง?**
**A:** AI ดู Pattern ไม่ยึด Layout กรณีอ่านไม่ได้ Flag เป็น Manual

**Q10: Scan คุณภาพไม่ดีทำยังไง?**
**A:** ระบบคัดกรองอัตโนมัติ ไม่กระทบ SLA

**Q11: เจ้าหน้าที่ต้องเรียนรู้เยอะไหม?**
**A:** ไม่มาก Flow งานเหมือนเดิม แค่ลดขั้นตอนอ่าน

## 4️⃣ IT / Architecture

**Q12: ต้องเปลี่ยนระบบ Claim เดิมไหม?**
**A:** ไม่ต้อง — ส่งข้อมูลเป็น JSON ผ่าน API เพียงปรับหน้าจอให้ ส่งรูป และรับข้อมูล จาก AI

**Integration กับระบบ Claim จะยุ่งยากไหม?**
**คำตอบ:**
“ไม่ยุ่งยากครับ
เพราะ Output เป็น JSON
เชื่อมกับระบบ Claim ได้เหมือนข้อมูลที่คนคีย์เอง
และเรามี Human Validation ขั้นกลาง
เพราะฉะนั้น Flow ปัจจุบันแทบไม่ต้องเปลี่ยนเลยครับ”

**Q13: ระบบล่มจะกระทบงานไหม?**
**A:** 
“ไม่มีผลกระทบต่อ Operation ครับ
ถ้า OCR ใช้งานไม่ได้ในบางช่วง
เคสเหล่านั้นจะกลับไปใช้ Flow Manual เหมือนเดิม
เพราะฉะนั้นไม่มีความเสี่ยงหยุดงานครับ”

**Q14: วัดความสำเร็จจากอะไร?**
**A:** ใช้ 3 ตัวชี้วัดนี้ คือ เวลา, ความถูกต้อง และ คำติชมจากผู้ใช้งาน
